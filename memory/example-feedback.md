This is an example feedback memory. Feedback memories capture corrections and confirmations so Claude doesn't repeat mistakes or drift from validated approaches.

Structure: lead with the rule, then Why (the reason), then How to apply (when/where it kicks in).

**Why:** Without feedback memories, you correct Claude once and it forgets by next session. With them, corrections stick permanently.

**How to apply:** When Claude does something wrong and you correct it, or when Claude does something right that wasn't obvious, save it here.
